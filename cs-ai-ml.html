Machine Learning

10 Key ML Algorithms

  1. Linear Regression: Fits a line/hyperplane to continuous output
  2. Logistic Regression: Binary classification via sigmoid function
  3. Decision Tree: Recursive feature splits; interpretable
  4. Random Forest: Ensemble of trees; reduces variance via bagging
  5. Gradient Boosting (XGBoost, LightGBM): Sequential error correction
  6. SVM: Maximises margin between classes; kernel trick for non-linear
  7. K-Nearest Neighbours: Classifies by majority vote of k nearest points
  8. K-Means: Clusters data by minimising within-cluster variance
  9. Naive Bayes: Probabilistic; assumes feature independence
  10. PCA: Dimensionality reduction via eigenvectors of covariance matrix

ML vs Deep Learning

Traditional ML: Requires manual feature engineering; works well on structured/tabular data with limited samples. Models are interpretable.

Deep Learning: Learns features automatically from raw data (images, text, audio); needs large datasets and compute. Models are less interpretable but more powerful for unstructured data.

Classification Evaluation Metrics

  • Accuracy: (TP+TN)/(TP+TN+FP+FN) — misleading on imbalanced datasets
  • Precision: TP/(TP+FP) — of all predicted positives, how many were correct?
  • Recall (Sensitivity): TP/(TP+FN) — of all actual positives, how many found?
  • F1 Score: 2 × (Precision × Recall)/(Precision + Recall) — harmonic mean
  • ROC-AUC: Area under the ROC curve; 1.0 = perfect, 0.5 = random

Regression Metrics

  • MAE: Mean Absolute Error — average |y − ŷ|; robust to outliers
  • MSE: Mean Squared Error — penalises large errors more heavily
  • RMSE: √MSE — same units as target; interpretable
  • R²: Proportion of variance explained; 1.0 = perfect fit
  • MAPE: Mean Absolute Percentage Error — scale-independent

Markov Chains

A stochastic process where the next state depends only on the current state (Markov property). Defined by a transition matrix P where Pij = probability of moving from state i to state j.

Applications: Language modelling, PageRank, MCMC sampling, hidden Markov models in NLP.

Shannon Entropy

Claude Shannon (1948). Measures the average information content (uncertainty) of a random variable X. Higher entropy = more uncertainty. Used in decision trees (information gain), compression, and cryptography.

Deep Learning

8 Steps to Train a Neural Network

  1. Prepare and preprocess training data
  2. Define model architecture (layers, activations, output)
  3. Choose loss function and optimiser
  4. Initialise weights (Xavier, He)
  5. Forward pass: compute predictions
  6. Compute loss between predictions and targets
  7. Backward pass (backpropagation): compute gradients
  8. Update weights using optimiser; repeat for each batch/epoch

Gradient Descent Update Rule

θ = parameters, η = learning rate, ∇θL = gradient of loss w.r.t. parameters. Variants: SGD, Adam, RMSProp, AdaGrad.

Backpropagation Chain Rule

Gradients flow backwards through the computational graph using the chain rule. Each layer's gradient is the product of all downstream gradients.

DQN Family

  • DQN: Deep Q-Network — experience replay + target network
  • Double DQN: Decouples action selection from evaluation to reduce overestimation bias
  • Dueling DQN: Separates value V(s) and advantage A(s,a) streams
  • Prioritised Experience Replay: Samples transitions proportional to TD error

Policy Gradient Methods

  • REINFORCE: Monte Carlo policy gradient; high variance
  • Actor-Critic (A2C): Actor updates policy; critic estimates value function as baseline
  • A3C: Asynchronous parallel workers; diverse experience
  • PPO (Proximal Policy Optimisation): Clips the policy update ratio to prevent large jumps; stable and widely used
  • SAC (Soft Actor-Critic): Maximum entropy RL; balances reward and exploration

Modern RL Concepts

  • Offline RL: Learning from a fixed dataset without environment interaction
  • GRPO (Group Relative Policy Optimisation): Compares outputs within a group to avoid needing a critic; used in DeepSeek-R1
  • DAPO (Decoupled Clip and Dynamic Sampling Policy Optimisation): Decouples clip ratios for better stability
  • RLVR (RL with Verifiable Rewards): Uses programmatically verifiable reward signals (math, code) instead of human preference labels

LLMs and AI Infrastructure

RAG vs Agentic RAG

Standard RAG: Query → retrieve docs → augment prompt → LLM generates answer. Single-shot retrieval.

Agentic RAG: LLM agent iteratively queries, reasons, reformulates queries, and synthesises across multiple retrieved documents. Handles complex multi-hop questions.

RAG Evaluation Metrics

  • Context Precision: Fraction of retrieved chunks that are relevant
  • Context Recall: Fraction of relevant information that was retrieved
  • Faithfulness: Whether the answer is grounded in the retrieved context (no hallucination)
  • Answer Relevance: Whether the answer addresses the question

5 LLM Fine-Tuning Techniques

  • LoRA: Low-Rank Adaptation — adds low-rank matrices to frozen weights; very parameter-efficient
  • LoRA-FA: Freezes matrix A during training; saves memory further
  • VeRA: Very-low-rank adaptation with shared frozen random matrices; even fewer parameters
  • Delta-LoRA: Updates the pre-trained weight using delta of LoRA products over training steps
  • LoRA+: Sets different learning rates for A and B matrices for better convergence

MCP vs A2A

MCP (Model Context Protocol): Standardises how LLMs connect to external tools and data sources. One model ↔ many tools.

A2A (Agent-to-Agent): Protocol for agents to communicate with other agents; enables multi-agent orchestration where specialised sub-agents collaborate.

LLM vs Agent vs Agentic Workflow vs Multi-Agent

  • LLM: Stateless language model; single prompt → single response
  • Agent: LLM + tools + memory; can take actions across multiple steps
  • Agentic Workflow: Structured sequence of LLM calls with conditional branching and tool use
  • Multi-Agent System: Network of specialised agents with defined roles; orchestrator coordinates sub-agents

AI System Architecture Layers

  1. Ingestion: Web scraping, APIs, databases, document parsers

Research Methods

Three-Pass Method for Reading Papers

  1. First pass (5–10 min): Title, abstract, section headings, conclusions. Decide if worth reading.
  2. Second pass (1 hr): Read with attention, skip proofs; understand figures, note key references.
  3. Third pass (4–5 hrs): Virtually re-implement the paper; challenge every assumption; identify gaps.

Feynman Technique

  1. Choose a concept and study it
  2. Explain it in simple language as if teaching a child
  3. Identify gaps in your explanation and go back to the source
  4. Simplify and use analogies until the explanation is clear and concise
